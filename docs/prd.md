## 체험단 매칭 플랫폼 MVP 제품 요구사항 명세서 (PRD)

### 1. 제품 개요 (Product Overview)

*   제품명: 체험단 매칭 플랫폼 (가칭)
*   비전: 광고주와 인플루언서를 위한 가장 직관적이고 효율적인 체험단 연결 솔루션.
*   제품 요약: 본 제품은 제품 및 서비스 홍보가 필요한 광고주와 새로운 경험을 원하는 인플루언서를 연결하는 웹 기반 플랫폼입니다. 광고주는 손쉽게 체험단을 등록하고 지원자를 관리할 수 있으며, 인플루언서는 다양한 체험단에 간편하게 지원하고 활동 상태를 추적할 수 있습니다.
*   MVP 목표:
    *   핵심 기능(체험단 생성, 지원, 선정)의 시장성을 검증한다.
    *   내부 베타테스트를 통해 초기 사용자(광고주, 인플루언서)로부터 핵심적인 피드백을 수집한다.
    *   오버엔지니어링을 지양하고 가장 빠른 속도로 동작하는 첫 번째 프로토타입을 완성하여 학습 속도를 극대화한다.

### 2. Stakeholders

*   CTO (Decision Maker): 기술 스택 선정, 개발 로드맵 수립 및 MVP 프로젝트의 최종 결과물에 대한 책임을 진다.
*   개발팀 (Executor): 본 PRD를 기반으로 실제 제품을 설계, 개발, 배포한다.
*   CEO/사업 개발 (Internal Customer): 내부 베타테스트에 참여할 초기 광고주 및 인플루언서 섭외를 담당하며, 비즈니스 관점의 피드백을 제공한다.
*   초기 베타테스터 (Advertisers, Influencers): 제품을 직접 사용하고 개선을 위한 핵심적인 피드백을 제공하는 최종 사용자 그룹.

### 3. 포함 페이지 (Pages in Scope)

MVP 단계에서는 아래의 핵심 페이지 개발에만 집중합니다.

| # | 페이지명 | 주요 기능 및 설명 |
|---|---|---|
| 1 | 홈 (Home) | - 서비스의 첫인상을 결정하는 랜딩 페이지.<br>- 상단 배너: 가장 최근에 등록된 체험단 1개를 고정 노출 (추천 로직 제외)<br>- 현재 모집 중인 전체 체험단 목록을 최신순으로 노출한다. |
| 2 | 회원가입 & 온보딩 | - 공통 정보(이름, 연락처 등) 입력 후 '광고주' 또는 '인플루언서' 역할을 선택.<br>- 역할에 따른 추가 정보(사업자번호, SNS 링크 등)를 입력하여 가입을 완료한다. |
| 3 | 체험단 상세 | - 캠페인의 상세 정보(혜택, 미션, 기간 등)를 확인하는 페이지.<br>- 인플루언서로 로그인 시 '지원하기' 기능이 활성화된다. |
| 4 | 체험단 지원 | - 인플루언서가 캠페인에 지원하기 위해 간단한 정보를 입력하고 제출하는 페이지. |
| 5 | 내 지원 목록 (인플루언서) | - 내가 지원한 체험단 리스트를 상태(신청완료, 선정, 반려) 순으로 정렬하여 노출 (별도 필터 UI 없음) |
| 6 | 체험단 관리 (광고주) | - 광고주가 자신이 등록한 모든 캠페인 목록을 확인하고 신규 캠페인을 등록하는 대시보드. |
| 7 | 광고주용 체험단 상세 | - 특정 캠페인에 지원한 인플루언서 목록을 확인하고, 모집을 마감하며, 최종 체험단을 선정하는 관리 페이지.<br>- 지원자 목록은 페이지 새로고침 시 갱신됨 (실시간 아님) |

---
MVP 범위 외 (Out of Scope):
*   사용자 간 실시간 채팅/메시징 기능
*   상세 검색 및 필터링 기능
*   리뷰 콘텐츠 자동 수집 및 리포팅 기능
*   결제 및 정산 시스템
*   관리자를 위한 백오피스(Django Admin으로 대체)

### 4. 사용자 여정 (User Journey)

#### 4.1. 인플루언서 (Influencer)

*   타겟 유저: 블로그, YouTube, Instagram 등 SNS 채널을 운영하며 다양한 제품/서비스 체험 기회를 찾는 개인.
*   목표: 쉽고 빠르게 원하는 체험단을 발견하고 지원하며, 선정 결과를 확인한다.

| 단계 | 사용자 행동 | 인터랙션 페이지 |
|---|---|---|
| 1 | 회원가입 | [회원가입 페이지] 이름, 연락처 등 기본 정보 입력 후 '인플루언서' 역할을 선택하고, SNS 채널 정보를 등록하여 가입을 완료한다. |
| 2 | 체험단 탐색 | [홈 페이지] 모집 중인 체험단 리스트를 둘러보다가 관심 있는 캠페인 카드를 클릭한다. |
| 3 | 정보 확인 | [체험단 상세 페이지] 제공 혜택, 미션, 모집 기간 등 상세 내용을 꼼꼼히 확인하고 참여를 결심한다. |
| 4 | 지원 | [체험단 상세 페이지] '지원하기' 버튼을 클릭하여 지원 절차를 시작한다. |
| 5 | 지원서 제출 | [체험단 지원 페이지] 각오 한마디, 방문 희망일 등 필수 정보를 입력하고 최종 제출한다. |
| 6 | 결과 확인 | [내 지원 목록 페이지] 지원 후, 해당 페이지에 접속하여 내 지원 상태가 '신청완료'에서 '선정' 또는 '반려'로 변경되었는지 스크롤하며 확인한다. |

#### 4.2. 광고주 (Advertiser)

*   타겟 유저: 오프라인 매장, 온라인 쇼핑몰, 중소 브랜드를 운영하며 마케팅 효율을 높이고자 하는 사업자.
*   목표: 최소한의 노력으로 체험단 캠페인을 생성하고, 지원자를 관리하여 홍보에 가장 적합한 인원을 선정한다.

| 단계 | 사용자 행동 | 인터랙션 페이지 |
|---|---|---|
| 1 | 회원가입 | [회원가입 페이지] 기본 정보 입력 후 '광고주' 역할을 선택하고, 업체명, 사업자등록번호 등 비즈니스 정보를 입력하여 가입을 완료한다. |
| 2 | 캠페인 생성 | [체험단 관리 페이지] 대시보드에서 '신규 체험단 등록' 버튼을 클릭하여 체험단 생성 다이얼로그 폼을 연다. |
| 3 | 정보 입력 | [체험단 관리 페이지] 체험단명, 모집 기간, 혜택, 미션 등 필수 정보를 모두 입력하고 캠페인을 등록한다. |
| 4 | 지원자 모니터링 | [광고주용 체험단 상세 페이지] 등록한 캠페인 상세 페이지로 이동하여, 페이지를 새로고침(F5)하며 새로운 지원자 목록을 확인한다. |
| 5 | 모집 마감 | [광고주용 체험단 상세 페이지] 모집 기간이 종료되면 '모집 종료' 버튼을 눌러 더 이상 지원자를 받지 않도록 상태를 변경한다. |
| 6 | 체험단 선정 | [광고주용 체험단 상세 페이지] 지원자 목록을 검토한 후, '체험단 선정' 버튼을 클릭하여 최종 인원을 선택하고 선정을 완료한다. |

### 5. 정보 구조 (Information Architecture - IA)

플랫폼의 전체적인 페이지 구조와 흐름을 시각화한 트리입니다. (변경 없음)

```
(Root)
|
├── / (Home) - (모든 사용자)
|
├── /accounts/
|   ├── /signup/ (회원가입 & 온보딩) - (비로그인)
|   ├── /login/ (로그인) - (비로그인)
|   └── /logout/ (로그아웃) - (로그인)
|
├── /campaigns/ (체험단)
|   ├── /<int:pk>/ (체험단 상세) - (모든 사용자)
|   └── /<int:pk>/apply/ (체험단 지원) - (인플루언서 전용)
|
├── /my/
|   └── /proposals/ (내 지원 목록) - (인플루언서 전용)
|
└── /manage/
    ├── /campaigns/ (체험단 관리 대시보드) - (광고주 전용)
    └── /campaigns/<int:pk>/ (광고주용 체험단 상세 및 지원자 관리) - (광고주 전용)
```
